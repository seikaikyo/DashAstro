# 宿曜道頁面 UI 重整

## 問題分析

目前頁面功能堆疊，視覺層次不清：

1. **資訊過載**：查詢一次出現本命宿 + 配對星宿 + 運勢，太多
2. **流程不明**：用戶不知道該先看哪裡
3. **空間浪費**：PC 版兩欄但配對區太擠
4. **重複元素**：四大運勢條在日/月/年都重複出現

---

## 目標

- 清晰的視覺層次
- 漸進式揭露資訊
- 每個區塊有明確目的
- 手機版也好用

---

## 方案選項

### 方案 A：Tab 分頁式

```
┌─────────────────────────────────────────┐
│ 宿曜道                                  │
├─────────────────────────────────────────┤
│ [生日輸入]           [查詢]             │
├─────────────────────────────────────────┤
│ [本命宿] [配對] [運勢] [相性診斷]        │ ← Tab
├─────────────────────────────────────────┤
│                                         │
│     各 Tab 內容                          │
│                                         │
└─────────────────────────────────────────┘
```

**優點**：資訊分區明確，不會同時看太多
**缺點**：需要切換，無法一眼看全部

### 方案 B：卡片摺疊式

```
┌─────────────────────────────────────────┐
│ 宿曜道                                  │
├─────────────────────────────────────────┤
│ [生日輸入]           [查詢]             │
├─────────────────────────────────────────┤
│ ┌─────────────────────────────────────┐ │
│ │ ▼ 本命宿：畢宿（日）              95│ │ ← 可展開
│ └─────────────────────────────────────┘ │
│ ┌─────────────────────────────────────┐ │
│ │ ▶ 今日運勢                       大吉│ │
│ └─────────────────────────────────────┘ │
│ ┌─────────────────────────────────────┐ │
│ │ ▶ 尋找配對                          │ │
│ └─────────────────────────────────────┘ │
│ ┌─────────────────────────────────────┐ │
│ │ ▶ 雙人相性                          │ │
│ └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

**優點**：漸進式揭露，預設只看摘要
**缺點**：需要手動展開

### 方案 C：Dashboard 總覽式

```
┌───────────────────────────────────────────────────┐
│ 宿曜道                   [生日] [查詢]            │
├───────────────────────────────────────────────────┤
│ ┌──────────────┐  ┌──────────────┐  ┌───────────┐│
│ │  本命宿      │  │  今日運勢    │  │  本月    ││
│ │  畢宿（日）  │  │     90      │  │    95    ││
│ │  [詳情→]    │  │   大吉      │  │   大吉   ││
│ └──────────────┘  └──────────────┘  └───────────┘│
├───────────────────────────────────────────────────┤
│ 尋找你的最佳配對                                  │
│ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│ │ 榮親 95  │ │ 業胎 90  │ │ 安壊 55  │          │
│ │ 角宿 亢宿│ │ 女宿 斗宿│ │ 房宿 心宿│          │
│ └──────────┘ └──────────┘ └──────────┘          │
└───────────────────────────────────────────────────┘
```

**優點**：一目了然，像儀表板
**缺點**：資訊密度高，可能還是覺得亂

---

## 建議方案

**方案 B（卡片摺疊式）** + 預設展開最重要的卡片

理由：
1. 漸進式揭露最適合「功能多但不一定每次都用」的情況
2. 用戶可自己選擇要看什麼
3. 手機版自然就是垂直堆疊
4. 摘要行讓用戶快速掌握重點
