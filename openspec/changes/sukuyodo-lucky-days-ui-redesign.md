# 吉日查詢 UI 重新設計

## 問題描述

目前吉日查詢介面存在以下問題：
1. **垂直空間過長**：9 個類別按鈕 + 多個項目按鈕 + 結果列表，導致頁面過長
2. **水平空間浪費**：左右兩側留白過多，沒有善用寬螢幕
3. **操作流程冗長**：需要多次點擊才能看到結果

## 設計決策

經用戶確認，採用以下方案：
- **類別選擇**：側邊欄 + 主內容區（左側固定類別列表，右側顯示項目和結果）
- **結果呈現**：吉日/避日雙欄並排（左側吉日、右側避日）
