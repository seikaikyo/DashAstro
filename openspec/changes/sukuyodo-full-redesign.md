# 宿曜道頁面全面重設計

## 問題描述

目前頁面採用垂直堆疊的 CollapsibleCard，存在以下問題：

1. **功能區塊過多**：10+ 個卡片垂直排列，頁面過長
2. **資訊層級不清**：所有功能同等重要，缺乏優先級
3. **水平空間浪費**：寬螢幕左右留白過多
4. **操作流程分散**：相關功能分散在不同位置

## 現有功能盤點

| 區塊 | 功能 | 使用頻率 |
|------|------|---------|
| 查詢區 | 輸入生日查詢本命宿 | 高（首次使用） |
| 本命宿 | 顯示查詢結果、性格、元素 | 高 |
| 尋找配對 | 六種關係的配對列表 | 中 |
| 今日運勢 | 當日運勢分數與建議 | 高 |
| 本週運勢 | 週運勢趨勢 | 中 |
| 本月運勢 | 月運勢與主題 | 低 |
| 本年運勢 | 年度運勢與月度趨勢 | 低 |
| 雙人相性 | 與特定對象的相性分析 | 中 |
| 吉日查詢 | 各類別吉日（已重設計） | 中 |
| 我的配對 | 與收藏對象的相性 | 中 |
| 二十七宿輪盤 | 視覺化展示 | 低 |
| 六種關係/七曜詳解 | 教學說明 | 低 |

## 設計決策

經用戶確認，採用以下方案：
- **整體結構**：Tab 分頁式（3-4 個主要 Tab）
- **核心資訊**：頂部固定摘要卡（本命宿 + 今日運勢分數）
- **運勢整合**：日/週/月/年用小 Tab 切換

## UI 規格

### 功能分組

| Tab | 包含功能 |
|-----|---------|
| **運勢** | 日運勢、週運勢、月運勢、年運勢（內部 Tab 切換） |
| **配對** | 尋找最佳配對、雙人相性診斷、我的配對關係 |
| **吉日** | 吉日查詢（側邊欄式，已完成） |
| **知識** | 六種關係詳解、七曜元素、二十七宿輪盤 |
